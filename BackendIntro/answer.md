Q1. Role of Frontend (FE)

Frontend is the part of web application where user can directly see and interact with the web browser.
It displays content like text,images,buttons using the technologies like HTML,CSS and Javascript to structure and style pages.
It handles interactions like clicking,scrolling,type and form submissions.
Sends request to backend using HTTTP and acts like bridge between user and backend server.

Q2. Role of Backend (BE)

Backend is the server side part of web application that manages data,logic,security.
It handles the request recieved  from the frontend.
It stores,retrives,updates and deletes data from database.
It also manages user authentication.

Q3. Business Logic

Business logic is the UI logic that how the system works acording to the business requirements.
Real-time examples:
1.E-commerce website
.Applying dicounts only the cart values exceeds a certain value.
.Calcuating final price with tax and shipping cost.
2.Banking application
.Preventing withdrawal if account balance is insufficient.

3.Online exam system
.Auto submitting test if time expires.
.Preventing multiple attempts if only one attempt is allowed.

Q4. Client–Server Model

Cilent-Server model is the  communcication architecture that divides tasks between cilents and servers.
Cilent:User facing application.It sends requests and display responses.Ex:chrome browser
Server:It process cilent requests and run in backend and manages data.Ex:Web servers
Communication process:Client sends a request to the server----->Server processes the request----->Server sends a response back to the client.

Q5. Three-Tier Architecture

Three-Tier architecture is divides a web application into 3separate layes to improve scalability and security.
Presentation layer:Handles user interaction(frontend).Uses React,HTML,CSS
Application layer:Handles business logic(backend).Uses Node.JS,Java
Database/Data layer:Manages database operation.mySQL,mangoDb,Oracle
This architecture is used for easy maintance and imporve secuirty.
